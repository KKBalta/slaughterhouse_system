# Reporting App - Detailed Design

This document details the design of the `reporting` Django app, which is responsible for generating comprehensive internal reports and analytics for management. It will aggregate data from various other applications to provide insights into KPIs, yield, throughput, and financial summaries.

## Core Models

### 1. `Report` Model

Defines a specific type of report that can be generated by the system.

*   **Purpose:** To manage different report configurations and their metadata.
*   **Key Fields:**
    *   `name` (CharField, unique): A descriptive name for the report (e.g., "Daily Throughput Report", "Monthly Yield Analysis").
    *   `description` (TextField, optional): Explains the purpose and content of the report.
    *   `report_type` (CharField with choices): Categorizes the report (e.g., 'OPERATIONAL', 'FINANCIAL', 'ANALYTICS').
    *   `configuration` (JSONField): Stores parameters and settings specific to how this report is generated (e.g., date ranges, filters, aggregation methods).
    *   `is_active` (BooleanField): Whether the report is currently available for generation.

### 2. `GeneratedReport` Model

Stores instances of generated reports, allowing for historical access and auditing.

*   **Purpose:** To archive generated reports and their associated metadata.
*   **Key Fields:**
    *   `report_definition` (ForeignKey to `Report`): The definition of the report that was generated.
    *   `generated_by` (ForeignKey to `users.User`): The user who generated the report.
    *   `generated_at` (DateTimeField): Timestamp of when the report was generated.
    *   `start_date` (DateField, optional): The start date of the data period covered by the report.
    *   `end_date` (DateField, optional): The end date of the data period covered by the report.
    *   `file_path` (FileField, optional): Path to the generated report file (e.g., PDF, CSV).
    *   `status` (CharField with choices): Status of the report generation (e.g., 'SUCCESS', 'FAILED').

## App Functionality

*   **Report Generation:** Provides mechanisms to generate various reports based on predefined configurations.
*   **Data Aggregation:** Pulls and aggregates data from `reception`, `processing`, `inventory`, and other relevant apps.
*   **Report Scheduling:** (Future) Ability to schedule reports to be generated automatically at specified intervals.
*   **Report Archiving:** Stores generated reports for historical access and auditing.
*   **Dashboard Integration:** (Future) Can provide data feeds or components for a management dashboard.
*   **User Interface:** Offers an interface for users to select, configure, and generate reports.
